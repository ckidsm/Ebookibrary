📚 설치 가이드

Tampermonkey 확장 + 교보 e-Library 동기화 Userscript 설치 — 한 번이면 끝.

교보문고 사이트는 SPA(Single Page App)라 일반적인 백엔드 자동 로그인이 어렵습니다. 대신 본인 브라우저의 로그인 세션을 그대로 활용하는 Tampermonkey 확장 + Userscript 방식을 사용합니다. 처음 한 번만 설치하면 이후로는 클릭 한 번으로 동기화 가능합니다.

지원 브라우저: Chrome, Edge, Safari (각각 Tampermonkey 공식 지원). Firefox는 Greasemonkey가 일반적이지만 Tampermonkey도 가능합니다.
1 Tampermonkey 확장 설치
  1. tampermonkey.net 접속
  2. 상단 탭에서 본인 브라우저 선택 (사진은 Chrome 탭 활성 상태)
  3. 아래 DOWNLOAD 섹션의 녹색 [Get from Store] 버튼 클릭 → Chrome Web Store로 이동
  4. Chrome Web Store에서 [Chrome에 추가] → 권한 확인 팝업의 [확장 프로그램 추가]
  5. 브라우저 우상단 퍼즐 조각 아이콘에서 Tampermonkey 검은 아이콘 보이면 설치 완료
BETA 카드는 최신 실험 버전. 일반 사용은 첫 번째 Tampermonkey 카드(Chrome 120+) 로 충분합니다.
2 개발자 모드 켜기 + 설치 확인

Tampermonkey가 외부 URL의 Userscript를 설치하려면 Chrome의 개발자 모드가 필요합니다.

  1. 주소창에 chrome://extensions/ 입력 → Enter
  2. 우상단 [개발자 모드] 토글 켜기 (켜져 있으면 좌상단에 [압축해제된 확장 프로그램 로드] 같은 버튼들이 나타남)
  3. 설치된 확장 카드 목록에 Tampermonkey 가 보이고 우하단 토글이 켜진(파란색) 상태인지 확인
Edge는 edge://extensions/, 좌측 하단 [개발자 모드] 토글입니다.

추가로 브라우저 우상단 Tampermonkey 검은 아이콘을 클릭하면 메뉴가 뜨고 "활성화됨" 상태로 표시되어야 합니다. 이 시점에는 "아무런 스크립트도 동작 중이지 않음"이 정상 — STEP 3 에서 우리 Userscript 가 추가됩니다.

3 Userscript 설치 (가장 중요한 단계)
⚠ Tampermonkey 홈의 [Userscripts] 탭에서 검색하지 마세요 — 우리 스크립트는 NAS 직접 URL이라 아래 큰 시안 버튼으로 한 방에 설치됩니다.

📜 sync-kyobo-library.user.js 바로 설치 →

  1. 위 시안 버튼 클릭 (새 탭) → Tampermonkey 설치 화면 자동 표시 (아래 캡처 참고)
  2. 좌측 [설치] 버튼 클릭 (우측은 [취소])
  3. 설치 완료 후 탭 자동 닫힘 또는 "Successfully installed" 표시
  4. Tampermonkey 아이콘 다시 클릭 → "1개 동작 중" 표시로 바뀌면 OK
설치 화면 확인 포인트 — 좌측 박스에 이름 Kyobo e-Library → NAS Sync, 버전 v0.2.0, 작성자 YUNDEOKSOO, 소스 http://192.168.10.205:8080/userscript/... 가 보이고 우측 포함 에 https://elibrary.kyobobook.co.kr/*, https://ebook.kyobobook.co.kr/dig/* 두 도메인이 들어 있어야 정상.
주소창의 chrome-extension://...ask.html?aid=... 가 보여도 정상 — 이게 별도 권한 요청 페이지가 아니라 위에 보이는 그 Tampermonkey 설치 페이지의 실제 URL입니다. 그대로 좌측 [설치] 만 누르면 끝.
이미 설치된 적이 있다면 "유저 스크립트 재설치" 화면이 뜨고 "모든 스크립트 설정이 재설정될 것입니다!" 안내(우상단 ⓘ 정보 아이콘)가 표시됩니다. 위험한 경고가 아니라 GM_setValue 로 저장한 백엔드 URL 설정 등이 기본값으로 돌아간다는 뜻 — 첫 설치라면 그대로 [재설치] 클릭.
4 교보 e-Library 로그인 + 동기화
  1. 홈으로 돌아가 [📖 내 e-Library] 카드 클릭 → 새 탭에서 교보 로그인
  2. 로그인 완료 후 도서함 페이지가 보이면 우측 하단에 📚 NAS Sync 다크 패널 자동 등장
  3. [미리보기] 클릭 → F12 → Console 탭에서 추출된 도서 확인
  4. 잘 추출됐으면 [동기화] 클릭 → "신규 N건 · 갱신 M건" 메시지 표시
  5. 홈으로 돌아와 새로고침 — [내 e-Library 도서함] 섹션에 카드 등장 ✨
한 번 설치하면 끝입니다. 이후로는 도서함 페이지 방문 → [동기화] 클릭으로 최신화.
! 잘 안 될 때
  • 다크 패널이 안 보임 — 페이지 로드 완전히 끝나길 기다리고 새로고침. Tampermonkey 우상단 아이콘 클릭해서 스크립트가 "활성화됨" 인지 확인.
  • ⚠ 카드 추출 실패 메시지 (아래 캡처처럼) — 교보 페이지 DOM 구조 차이. F12 → Console 의 [NVK] 진단용 로그 캡처 후 개발자에게 알려주면 셀렉터 보정 가능. v0.3.0 부터는 진단 dump 가 훨씬 풍부합니다.
  • ✗ 네트워크 오류 — NAS Kyobo Bridge(9000) 컨테이너 다운. http://192.168.10.205:9000/health 직접 호출해 응답 확인.
  • 로그인이 자꾸 풀림 — 교보 세션 만료. 다시 로그인 후 [동기화] 재시도.
  • Tampermonkey 홈 → [Userscripts] 탭으로 가서 우리 스크립트를 검색하지 마세요 — 이 탭은 GreasyFork·OpenUserJS 같은 외부 사이트 안내일 뿐이고, 우리 스크립트는 NAS 서버 직접 URL로 설치합니다 (STEP 3 의 녹색 버튼). 아래 화면을 만났다면 그냥 닫고 STEP 3 으로 돌아가세요.
