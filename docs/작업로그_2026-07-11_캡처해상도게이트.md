외장 Type-C 모니터에 교보 데스크탑 앱을 띄워 캡처하려 했는데, **외장 모니터의 HiDPI(고해상) 설정에 따라 캡처 페이지당 픽셀이 달라지는데도 시스템이 이를 감지·판정하지 못했다.**
- **원인 1 (도구가 안 돎)**: `book-capture/.venv`가 깨져(OneDrive가 다른 Mac의 venv를 10바이트 스텁으로 만듦) 모니터 해상도/scale을 읽는 도구(Quartz) 자체가 조용히 no-op. → 외장이 HiDPI인지 1x인지조차 못 읽음.
- **원인 2 (검증·경고 없음)**: 웹(도서 라이브러리)에서 로컬 매크로로 분석을 시작해도 **캡처 대상 화면이 표준(양면 페이지당 ≥1400px)을 만족하는지 아무 검증·경고가 없었다.** 그래서 외장이 미달이어도(1280×800 HiDPI=양면 1280px, 1920×1200=960px) 그대로 저해상 캡처가 됐다.
- 요컨대 **"외장 HiDPI 설정을 제대로 잡지 못해 저해상 캡처로 이어지는"** 문제.

## 4. 요구사항 (사용자 지정)
1. 로컬 매크로 선택 후 분석 시 — 캡처 대상 화면이 양면 표준(페이지당 ≥1400px) 미달이면 **경고/오류**.
2. 외장 사용은 **강제 아님**(멀티태스킹하며 외장 캡처는 정당).
3. 외장 미달이면 → 권장 해상도 제시 + 직접 변경 안내(코드가 시스템 해상도 자동 변경 안 함).

**결정**: 미달 시 = **차단 + 조치 안내**(`그래도 진행`으로만 우회). 해상도는 **권장 제시 + 직접 변경**.

## 5. 소스 수정 상세

### 5-1. `book-capture/bookcapture/capture_standard.py` — 규칙화 클래스 (추가)
기존(순수 상수·함수) 뒤에 클래스 2개 추가. Quartz 비의존(순수 계산) → Win/Linux에서도 import 가능.

- **`class DisplaySpec`** — 모니터 한 대의 캡처 능력:
  - `backing_width` = `round(width_pt * scale)` (최대화 시 캡처 폭)
  - `page_px(pages_per_spread=2, coverage=1.0)` = `width_pt*scale*coverage/페이지수`
  - `meets(pages_per_spread=2)` = `page_px >= MIN_SOURCE_WIDTH(1400)`
  - `best_layout()` → `"spread"`(양면 가능) / `"single"`(단면만) / `None`(불가)
  - `kind()` → `"내장"`/`"외장"`
- **`class CaptureStandardV1`** — 규칙 엔진:
  - `MIN_PAGE_PX = 1400`, `DEFAULT_PAGES_PER_SPREAD = 2`(교보 앱 기본 양면)
  - `required_backing_width(pages=2)` = `1400*pages` (양면 2800)
  - `evaluate(disp, pages=2)` → dict(page_px, single_page_px, meets, meets_single, need_backing, best_layout, advice[])
  - `plan(displays, pages=2, prefer_id=None)` → 여러 모니터 중 **캡처 추천 결정**:
    - `prefer_id`(사용자가 앱 띄운 모니터)가 충족이면 그대로 존중(**외장 강제 아님**)
    - 아니면 충족 모니터 중 페이지당 픽셀 최고(동률이면 내장 우선)
    - 아무도 양면 미달이면 `any_meets=False, override_needed=True` + 조치 안내(단면 가능 화면의 **단면 픽셀**, 해상도 상향, 내장 Retina)
