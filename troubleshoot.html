🪟 Windows 워커

원클릭 설치 / 갱신

PowerShell을 열고 한 줄 — Python 확인·설치 → 최신 코드 → 워커 자동등록까지:

irm https://redcodeme.synology.me/kyobo/setup-windows.ps1 | iex

⚠️ Python이 방금 설치됐다면 PowerShell을 새로 열고 다시 실행. 워커는 죽어도 5초 내 자동 부활(세션 켜둔 동안).

⚡ 한 방 진단·복구 (멈춤 의심 시 먼저)

$t = Get-ScheduledTask -TaskName KyoboBookcaptureWorker -ErrorAction SilentlyContinue; if (-not $t) { Write-Host "워커 미설치 - 위 원클릭 실행" -ForegroundColor Red } else { $i = $t | Get-ScheduledTaskInfo; Write-Host "상태: $($t.State) / 마지막실행: $($i.LastRunTime)"; if ($t.State -ne 'Running') { Start-ScheduledTask -TaskName KyoboBookcaptureWorker; Write-Host "-> 시작함" -ForegroundColor Green } else { Write-Host "-> 실행 중(정상)" -ForegroundColor Green } }

명령

상태 확인

Get-ScheduledTask -TaskName KyoboBookcaptureWorker | Get-ScheduledTaskInfo | Select State, LastRunTime, LastTaskResult

켜기 / 로그

Start-ScheduledTask -TaskName KyoboBookcaptureWorker
Get-Content "$env:LOCALAPPDATA\kyobo-library\worker.out.log" -Tail 40

🍎 Mac 워커

원클릭 설치 / 갱신

터미널에 한 줄 — 최신 코드 다운로드 → venv·의존성 → launchd 자동시작:

curl -fsSL https://redcodeme.synology.me/kyobo/setup-mac.sh | bash

python3 필요(없으면 brew install python@3.13). 로그인 시 자동시작(launchd).

명령

상태 확인 / 재시작 / 정지

launchctl list | grep kyobolibrary
launchctl kickstart -k gui/$(id -u)/com.kyobolibrary.worker
launchctl bootout gui/$(id -u)/com.kyobolibrary.worker # 정지

Mac은 deep link 로 책 자동 열기 지원. 브라우저 웹뷰어 캡처도 동일하게 동작.

🐧 Ubuntu / Linux 워커 (X11)

원클릭 설치 / 갱신

터미널에 한 줄 — scrot·xdotool·코드 설치 → systemd 자동시작:

curl -fsSL https://redcodeme.synology.me/kyobo/setup-linux.sh | bash

⚠️ X11 GUI 세션 필요(Wayland는 미지원). 화면 캡처라 노트북 GUI에 로그인 + 교보 뷰어가 화면에 떠 있어야 합니다. 죽어도 5초 내 자동 재시작.

명령

상태 / 재시작 / 로그

systemctl --user status kyobo-worker
systemctl --user restart kyobo-worker
journalctl --user -u kyobo-worker -f

DISPLAY가 :1 이 아니면 ~/.config/systemd/user/kyobo-worker.service 의 Environment=DISPLAY= 수정 후 systemctl --user daemon-reload && systemctl --user restart kyobo-worker.

🔍 상황별 해결 (공통)

Q. [분석 시작] 눌러도 아무 일도 안 일어난다

→ 워커가 꺼짐. 본인 OS 탭의 [상태 확인]/[한 방 진단] → 켜고 다시 시작.

Q. 워커가 자꾸 멈춘다

→ 워커는 화면을 캡처하므로 세션이 살아있어야 합니다(원격 끊김·절전 시 멈춤, 돌아오면 자동 부활).

Q. 여러 PC에 워커가 켜져 있다

→ 아무 워커나 잡을 수 있습니다. 캡처할 그 PC만 워커를 켜두고 나머지는 끄세요(안 그러면 엉뚱한 화면을 찍음).

Q. [바로보기] 버튼이 안 보인다

→ 교보 로그인 안 됨. 포털 [교보문고 로그인] 후 새로고침(로그인+소유해야 버튼 표시).

Q. 엉뚱한 화면이 캡처됐다

→ 캡처 순간 교보 뷰어가 맨 앞에 없었던 것. [준비 완료] 후 5초 안에 뷰어로 전환, 캡처 끝까지 다른 창 누르지 마세요.

Q. 양쪽(2페이지)으로 찍힌다

→ 뷰어 단일 페이지(1) 로 바꾸세요(거터·여백 없이 한 장씩).

✅ 깨끗한 캡처 순서 (공통)

  1. 본인 OS 워커 실행 중 확인 (위 탭)
  2. 교보 로그인 → 책 [바로보기] → 웹뷰어 열기
  3. 단일 페이지(1) + F11 전체화면 + 첫 페이지
  4. 포털 → 책 → [분석 시작] → 🌐 브라우저 웹뷰어 캡처 → [준비 완료]
  5. 5초 안에 뷰어로 전환 → 손 떼고 기다리기 (세션 끄지 말기)
