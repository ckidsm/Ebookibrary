캡처가 안 될 때 상황별 해결 + Windows 워커/유저스크립트 명령 모음

⚡ 한 방 진단·복구 (Windows)

제일 먼저 이걸 PowerShell에 붙여넣으세요. 워커 상태를 점검하고, 멈췄으면 자동으로 켭니다.

$t = Get-ScheduledTask -TaskName KyoboBookcaptureWorker -ErrorAction SilentlyContinue; if (-not $t) { Write-Host "워커 미설치 - 아래 [재설치] 명령 실행" -ForegroundColor Red } else { $i = $t | Get-ScheduledTaskInfo; Write-Host "상태: $($t.State) / 마지막실행: $($i.LastRunTime) / 결과코드: $($i.LastTaskResult)"; if ($t.State -ne 'Running') { Start-ScheduledTask -TaskName KyoboBookcaptureWorker; Write-Host "-> 워커를 시작했습니다" -ForegroundColor Green } else { Write-Host "-> 워커 실행 중 (정상)" -ForegroundColor Green } }

결과가 실행 중(정상) 이면 OK. 미설치 면 아래 [워커 재설치]를, 켰는데도 안 되면 [로그 확인]을 보세요.

🖥 Windows 워커 명령

① 상태 확인

Get-ScheduledTask -TaskName KyoboBookcaptureWorker | Get-ScheduledTaskInfo | Select State, LastRunTime, LastTaskResult

② 워커 켜기 (멈췄을 때)

Start-ScheduledTask -TaskName KyoboBookcaptureWorker

③ 로그 확인 (왜 죽었나)

Get-Content "$env:LOCALAPPDATA\kyobo-library\worker.out.log" -Tail 40

④ 워커 위치 찾기

(Get-ScheduledTask -TaskName KyoboBookcaptureWorker).Actions[0].WorkingDirectory

⑤ 워커 재설치 대부분의 문제 해결

설치 위치를 자동으로 찾아 재설치합니다. (먼저 OneDrive 동기화 완료 확인)

$bc = (Get-ScheduledTask -TaskName KyoboBookcaptureWorker).Actions[0].WorkingDirectory; Write-Host "위치: $bc"; powershell -ExecutionPolicy Bypass -File "$bc\scripts\install-worker-windows.ps1"

⑥ 워커 제거

$bc = (Get-ScheduledTask -TaskName KyoboBookcaptureWorker).Actions[0].WorkingDirectory; powershell -ExecutionPolicy Bypass -File "$bc\scripts\uninstall-worker-windows.ps1"

🧩 Tampermonkey 유저스크립트

① "사용자 스크립트 허용" 켜기 ★필수★

이게 꺼져 있으면 스크립트를 설치해도 아예 안 됩니다 (Chrome 최신 요구사항).

  1. 주소창에 chrome://extensions 입력
  2. 우측 상단 개발자 모드 토글 ON
  3. Tampermonkey → 세부정보 → 아래로 → "사용자 스크립트 허용" 토글 ON

② 설치 / 업데이트 (v0.8.0+)

포털 [📥 내 e-Library 동기화] 카드 → [이 Userscript 설치], 또는 직접:

https://redcodeme.synology.me/kyobo/userscript/sync-kyobo-library.user.js

이미 설치돼 있으면: Tampermonkey 대시보드 → 스크립트 → 마지막 업데이트 확인 으로 v0.8.0 으로 갱신 (로그인 감지 기능 포함).

🔍 상황별 해결

Q. [분석 시작] 눌러도 아무 일도 안 일어난다

→ 워커가 꺼졌을 가능성. 위 [한 방 진단·복구] 실행 → 워커 켜고 다시 [분석 시작].

Q. 워커가 자꾸 멈춘다 / 재설치해도 또 빠진다

→ 워커는 화면을 캡처하므로 Windows 세션이 살아있어야 합니다. 원격 데스크톱을 완전히 끊거나(로그오프)·PC 절전 되면 멈춥니다. 캡처하는 동안 세션을 켜둔 채 두세요. (세션 돌아오면 5초 내 자동 부활)

Q. 교보에서 [바로보기] 버튼이 안 보인다

→ 교보 로그인 안 됨. 로그인+소유해야 버튼이 나타납니다. 포털 상단 [교보문고 로그인] 후 새로고침.

Q. 엉뚱한 화면(바탕화면·다른 창)이 캡처됐다

→ 캡처 순간 교보 웹뷰어가 맨 앞에 없었던 것. [준비 완료] 후 5초 안에 웹뷰어로 전환(Alt+Tab) 하고, 캡처 끝날 때까지 다른 창 누르지 마세요.

Q. 양쪽(2페이지)으로 찍힌다 / 가운데 여백이 크다

→ 뷰어가 2페이지 펼침 모드. 뷰어 상단 단일 페이지(1) 로 바꾸거나(권장), 창을 세로로 하면 한 장씩 찍힙니다.

Q. PowerShell에 여러 줄 붙여넣으니 > 만 뜨고 안 된다

→ Enter 한두 번 누르거나, Ctrl+C 후 명령을 세미콜론(;)으로 이어 한 줄로 붙여넣으세요 (이 페이지 명령은 모두 한 줄로 준비됨).

Q. Tampermonkey에 스크립트가 없다 (Windows)

→ 위 [유저스크립트] ①② 순서대로: "사용자 스크립트 허용" 켜고 → 설치. Mac에서 됐어도 PC마다 따로 설치해야 합니다.

✅ 깨끗한 캡처 순서 (요약)

  1. 워커 실행 중 확인 (위 한 방 진단)
  2. 교보 로그인 → 책 [바로보기] → 웹뷰어 열기
  3. 단일 페이지(1) + F11 전체화면 + 첫 페이지
  4. 포털에서 책 → [분석 시작] → 🌐 브라우저 웹뷰어 캡처 → [준비 완료]
  5. 5초 안에 웹뷰어로 전환 → 손 떼고 기다리기 (세션 끄지 말기)
